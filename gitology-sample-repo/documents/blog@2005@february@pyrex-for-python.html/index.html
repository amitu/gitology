The whitepaper I talked about yesterday, also lead me to pyrex, which it introduces as:
Pyrex is a dialect of Python that is translated directly into C code. It's by far the most pleasant way to write C extensions for Python. Examining the Python code in detail revealed that the most expensive part of the calculation was the RMS - it was responsible for around 65% of the time taken. Moving just the RMS operation to Pyrex reduced that component from around 1.4ms to just 0.35ms.

The normal way of wrapping C libraries for Python is very cumbersome, and things like SWIG also are only partially helpful, pyrex looks like the best alternative.
